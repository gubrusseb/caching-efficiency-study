О проекте

Данный веб-сайт разработан в рамках дипломной работы для исследования влияния кеширующего слоя на быстродействие веб-приложений. Проект представляет собой интернет-магазин с типичной функциональностью, но с возможностью переключения режимов работы кеша для сравнительного анализа.

Цель исследования:

Оценить эффективность внедрения Redis как кеширующего слоя между приложением и базой данных PostgreSQL, измерить улучшение времени отклика и снижение нагрузки на БД.

Технологический стек

  • Kotlin — современный язык программирования для JVM
  • Spring Boot — фреймворк для создания микросервисов
  • PostgreSQL — основная реляционная база данных
  • Redis — кеширующий слой в оперативной памяти
  • Spring Cache — абстракция для работы с кешем
  • Bootstrap 5 — адаптивный дизайн интерфейса

Почему Redis?

Redis выбран как решение для кеширования благодаря:

  • Хранение данных в оперативной памяти
  • Поддержка сложных структур данных
  • Атомарные операции
  • Встроенная репликация и персистентность
  • Низкая задержка (субмиллисекундная)
  • Простая интеграция со Spring Boot
